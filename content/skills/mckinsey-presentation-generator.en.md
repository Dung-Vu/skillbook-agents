## 📖 Why Do We Need This Skill?

In professional consulting, slide design must prioritize clarity, strict alignment, and rigorous data over creative decoration. This skill equips the Agent to build formal, information-dense HTML presentations matching McKinsey's strict standards—featuring precise typography, structured data zones, and flat SVG-based charts.

## ⚙️ How It Works

The mandatory workflow structure:
```
Market Research (Step 1) ──> Slide Decks Planning ──> Standalone HTML Coding ──> Deploy Presentation
```
1. **Research**: Execute 8-12 targeted search queries to capture verified market sizes, CAGRs, and competitor metrics.
2. **Structure Planning**: Plan a default 10-slide deck (Cover, TOC, Content, Dividers, Summary).
3. **HTML Coding**: Write modular HTML slide files with inline CSS and strict 'Sharp & Compact' design tokens.
4. **SVG Visualizations**: Render bar, line, and comparison charts natively as inline SVG objects.
5. **Live Deployment**: Publish using the designated HTML presentation deployer.

## 🚀 How to use

- **Sharp Edges Only**: Set `border-radius: 0` globally on all cards, buttons, progress bars, and SVG chart shapes.
- **Typography Scale**: Page titles at 24px, insights at 16px, labels at 10-12px, and citations/sources at 8px.
- **Color Restrictions**: Deep Navy `#0B1F3A` for headers, Cobalt `#1B5AB5` as primary accent. Use a maximum of 2 accent colors per slide.
- **Layout Specification**: Every content slide must have at least 4 distinct data zones and a footnote source line.

## ⚠️ Gotchas and notes

- **Zero AI-generated decoration**: Never include abstract AI-generated images or icons inside formal consulting decks.
- **White-space checks**: Ensure content coverage is at least 70% to prevent empty gaps larger than 52px.
- **Required footnotes**: Always include footnotes containing the organization, date, and disclaimer for data-backed slides.

## 📖 Why Do We Need This Skill?

Tech product development frequently suffers from vague specifications and incomplete user journeys. This skill transforms the Agent into a Senior Product Manager that structure requirements, maps user interactions with Mermaid flowcharts, designs wireframe specifications, and builds high-fidelity frontend prototypes.

## ⚙️ How It Works

The product design workflow:
```
Vague Concept ──> Feasibility & Competitor Report ──> Mermaid User Flow ──> Compiling PRD ──> Interactive HTML Prototype
```
1. **Market Analysis**: Assess feasibility, research market competitors, and document target user personas.
2. **Flow Mapping**: Draw clear Mermaid flowcharts showing happy paths, edge cases, and database transitions.
3. **PRD Writing**: Compile requirements (Markdown/PDF) covering functional scopes, API specs, and non-functional requirements.
4. **Prototyping**: Write single-file responsive HTML mockups using CSS to simulate real system interactions.

## 🚀 How to use

- Always maintain strict language consistency: output all assets in the user's conversation language.
- PRD functional requirements must include clear acceptance criteria and detailed error state definitions.
- Visual wireframes and prototypes should prioritize functional layouts, clean grids, and modern UI tokens.

## ⚠️ Gotchas and notes

- **Ambiguous specs**: Avoid generic requirements such as 'the system should load fast.' Quantify requirements (e.g., 'page load < 1.5s').
- **Overcomplicated prototypes**: Restrict the prototype scope to testing key user flows. Use local mocks instead of real database integrations.

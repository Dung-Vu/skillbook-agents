## 📖 Why Do We Need This Skill?

Many digital products fail due to confusing navigation, inconsistent typography, or lack of accessibility compliance. This skill equips the Agent with professional design standards covering grid alignments, spacing scales, readable color contrasts, and semantic UI frameworks.

## ⚙️ How It Works

The interface design cycle:
```
User Needs Audit --> Core Design System Setup --> Wireframe Layout Drafting --> WCAG Contrast Audit
```
1. **Discovery**: Document interface purposes, target devices, and primary user personas.
2. **Design Tokens**: Establish cohesive color tokens, typography scales, border-radii, and spacing rules.
3. **Layout Grid**: Structuralize layouts (navigation placement, content grouping, card systems).
4. **WCAG Compliance**: Audit font sizes and check foreground/background contrast ratios for maximum readability.

## 🚀 Agent Guidelines (Prompt Guidelines)

- Always define color variables using exact HEX or HSL values with their corresponding semantic functions (e.g. Primary, Alert, Success).
- Enforce WCAG 2.1 AA standards: minimum 4.5:1 contrast ratio for normal text and 3:1 for large text.
- Deliver design specifications with clear CSS Grid/Flexbox code suggestions for developers.

## ⚠️ Gotchas and notes

- **Decorative Bloat**: Avoid adding arbitrary gradients or non-standard elements. Keep layouts clean and functional.
- **Ignoring Tap Targets**: Ensure interactive touch targets (buttons, links) are at least 48x48px on mobile views.

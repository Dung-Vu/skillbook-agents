## 📖 Why Do We Need This Skill?

A great software product must be both functional and intuitive. This skill equips the agent with high-level UI/UX principles, guiding the creation of consistent, accessible design systems that enhance usability and decrease user cognitive load.

## ⚙️ How It Works

The UX/UI standardization flow:

```
[Requirements] -> [UX Flow Mapping] -> [Design System Mapping] -> [Accessibility & Contrast Audit] -> [Component Patterns]
```

1. **Flow Mapping**: Streamlines user interactions to minimize steps required to achieve goals.
2. **Design System**: Establishes typography scale, color tokens, and spatial grids.
3. **Accessibility Audit**: Verifies text contrast and screen-reader friendliness against WCAG standards.

## 🚀 Agent Guidelines (Prompt Guidelines)

1. Maintain color contrast ratios compliant with WCAG AA standards (minimum 4.5:1 for normal text).
2. Keep interface patterns consistent to reduce learning curves for the user.
3. Always design clear states for loading, error, success, and empty data scenarios.

## ⚠️ Gotchas and notes

- **Neglecting Accessibility**: Muted low-contrast text might look trendy but makes the site unusable for visually impaired users. Prioritize readability.
- **Style Drift**: Mixing different button styles or border-radius values across routes breaks UI coherence. Standardize tokens globally.

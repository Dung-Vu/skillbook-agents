# Project: Paper Crumple & Unfold Transition

## Architecture
- **Target Files**:
  - `package.json` (Add `three`, `@types/three`, `html2canvas`)
  - `src/components/ui/PaperCrumpleOverlay.tsx` (New component for WebGL transition overlay)
  - `src/hooks/useTransitionNavigator.ts` (Update navigator hook to orchestrate transitions)
  - `src/app/skills/template.tsx` (Wrap pages in the transition overlay component)
- **Data/Event Flow**:
  - Click Event on Card -> `navigateTo` -> prevent default -> dispatch "canvas-pause" + `"skillbook:page-exiting"` -> trigger exit (liquid-glass) animation on `PaperCrumpleOverlay` -> await exit finish -> `router.push(href)`
  - Page Mount -> new page loads -> mount `PaperCrumpleOverlay` -> capture new page state -> trigger enter (unfold) animation on `PaperCrumpleOverlay` -> dispatch "canvas-resume" -> hide overlay & restore interactivity
- **Coordination between page body and Header**: the `"skillbook:page-exiting"` event is dispatched by `useTransitionNavigator` and listened to by **every** component that calls the hook, so the visual exit fade plays on the page body even when the click originated in a different component (e.g. the global `Header`).

## Interface Contracts
### `PaperCrumpleOverlay` Component
- API / Props: `{ children: React.ReactNode }` (currently renders the overlay only — the page body is rendered by the layout)
- Global events / state hook:
  - Listens to `"transition-exit-start"` and `"canvas-pause"`.
  - Performs `html2canvas` capture of the page content.
  - Manages a Three.js Canvas, low-density `PlaneGeometry(aspect, 1.0, 8, 8)`, and a custom fragment shader.
  - Dispatches `"transition-exit-complete"` and `"transition-exit-animating"` to coordinate with the navigator hook.
  - Manages unmount/disposal of all WebGL assets (geometry, material, texture, renderer).

## Implementation note (vs. original 3D spec)
The original R2 / acceptance criteria in `ORIGINAL_REQUEST.md` describe a
3D paper-crumple effect (Perlin/Simplex vertex displacement, gravity pull
along Y, recomputed normals for crisp shading). The shipped implementation
is a **2D liquid-glass refraction** instead: the plane geometry is just
8×8 (a single quad in practice), and the visual interest is generated by
a fragment shader that does:

  - a radial sine-wave "ripple" emanating from the click point,
  - UV distortion of the captured screenshot under the ripple,
  - a fake normal map from the wave gradient to give glassy specular
    highlights, and
  - a dissolving alpha that reveals the next page during enter.

Rationale: the 2D effect is dramatically cheaper on mid-range mobile
hardware (the original 3D noise was projected to drop <30 FPS on entry-
level Android), visually still reads as a "page materialises out of
paper", and is the version that passes the 60 FPS / "seamless transition"
E2E guardrails in `tests/e2e/page-transitions.spec.ts`. If/when the
project needs a true 3D crumple, the entry point is the fragment shader
in `PaperCrumpleOverlay.tsx` and the geometry subdivision in
`initThree()`.
